# Judge 的进化机制（借鉴 darwin-skill）

VideoBench 里被评估的不只有视频——**Judge 本身也是被评估和进化的对象**。本文件借鉴 darwin-skill 的"度量-保留"式进化（measure-and-preserve），规定 Judge prompt/rubric 如何安全地迭代。

## 核心机制

### 1. 单一可变资产 + 单一变量迭代

- 每个 Judge 的可变资产就是它的 prompt 文件（`prompts/*.md`）与维度定义（`docs/dimensions.md`）；
- 每次迭代只改**一个东西**：一个 prompt 的一处策略、或一个维度的锚点描述。一次改多处，黄金集分数变了也无法归因。

### 2. 棘轮机制（Ratcheting）

Judge 的"分数"= 在人类黄金集上的表现（calibration.md 的六项检验，核心是 GSB 三分类一致率与 Gate 致命召回率）：

1. 改动前记录当前版本的黄金集成绩（基线）；
2. 改动后重跑黄金集，得到新成绩；
3. **提升才保留，持平或下降立即回滚**——分数只许向上走；
4. 每次保留的改动记录版本号 + 成绩 + 改了什么，追加到 `runs/judge-versions.md`。

这与本机的 golden-set-regression skill 是同一思想："跑考卷，防止修好一个弄坏三个"。

### 3. 双重评估

darwin-skill 用"静态结构分析 + 运行时功能验证"双轨评估 skill；对应到视频评估：

- **静态轨（客观检查）**：ffprobe 规格、OCR 错字比对、ASR 语言/事实对照、LUFS 响度、帧间差分——确定性工具，结果不容 Judge 推翻；
- **运行时轨（Judge 主观评估）**：美学、叙事、运动自然等工具测不了的维度。

分工的唯一权威在 dimensions.md 的"评测方式"列。客观轨结果与 Judge 结论冲突时，以客观轨为准并记一次 Judge 误判（进误判分析）。

### 4. 独立评审，禁止自评

LLM 自评有严重偏差（darwin-skill 引用的 SkillLens 数据：自评准确率仅 46.4%）。因此：

- 生成视频的 agent/模型不得评自己产出的视频；
- 优化 Judge prompt 的 agent 不得自己宣布"改进有效"——必须由独立运行的黄金集回归给出数字；
- 高价值评估用多 Judge 投票，投票成员应有模型或证据配置上的多样性。

### 5. 人类检查点

不追求全自动。以下节点强制人工确认：

- Judge prompt 新版本上线前（看黄金集成绩 + 抽查误判样本）；
- Gate 规则的增删改（漏放致命是最重故障）；
- 黄金集本身的扩充与修订（人类标注是整个体系的地基）。

## 反模式清单（禁止）

1. 不给证据的总分——用一个笼统分数掩盖维度差异；
2. 生成者自评，或 Judge 优化者自己宣布优化有效；
3. 一次改多个 prompt/rubric 后声称"整体变好了"；
4. 没跑黄金集回归就上线新 Judge 版本；
5. 把随机波动解释成提升或退化（不允许 S、或在个位数样本上下结论）；
6. 让美学分覆盖任务完成度和事实错误;
7. 用稀疏抽帧评时序类维度还照常给分；
8. 报告里隐藏 Same，只报 Decisive Win Rate；
9. 用引导性措辞修复错杀（直接告诉 Judge"某某不算缺陷、不要苛刻"）而不做双向回归——错杀会翻转成放水，且在没有负样本的黄金集上不可见（2026-07-15 实测：加了"类型语言不是缺陷"后同模型从 1.5 分跳到 5.0 分，是否过校正无法判定）。正确做法：给中性的类型上下文 + 对标锚点（"在同类型商业作品中处于什么档位"），并在含差样本的黄金集上同时测错杀率与放水率；
10. 一份 prompt 打天下——不同类型视频必须经 genre router 路由到对应 rubric 与观众人格。
