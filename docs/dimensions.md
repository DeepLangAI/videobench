# 六层质量维度 Taxonomy

本文件是 VideoBench 评估维度的唯一权威定义。任何 Judge prompt、schema、报告口径都必须与此对齐。

评估对象只有一个：产出的成片。模型对每条视频形成质量向量 Q(y) = [q_1, q_2, …, q_n]，各分量按以下六层组织，每维 1–5 分并附证据。

三个标记：

- **⚑ 致命候选**：该维度发现严重问题时直接进入 Gate 判定（见 protocol.md）；
- **评测方式**：`客观`（工具确定性测量，不走 Judge）/ `Judge`（主观判断）/ `混合`（工具先测、Judge 复核）——能客观测的绝不交给 Judge；
- **需参照**：该维度依赖 Brief/产品事实/平台/品牌参照，无参照模式下不可评。

## L1 任务层（Task Alignment）

回答：用户要的东西做出来了吗？

| 维度 | 定义 | 评测方式 | 需参照 |
|---|---|---|---|
| 指令遵循 | 是否覆盖 Brief 的显式要求（主题、结构、指定素材、指定镜头等） | Judge | ✓ |
| 信息准确 ⚑ | 产品事实、数字、名称、卖点表述与产品事实表一致 | 混合：OCR/ASR 提取后与事实表逐条比对，冲突项交 Judge 确认语义 | ✓ |
| 内容完整 ⚑ | 关键内容（核心卖点、必要免责声明、CTA）无缺失 | 混合：must_appear 事实规则检出，其余 Judge | ✓ |

## L2 视觉层（Visual Quality）

| 维度 | 定义 | 评测方式 | 需参照 |
|---|---|---|---|
| 清晰度 | 分辨率、压缩伪影、噪点、模糊 | 混合：分辨率/码率客观，感知糊不糊 Judge | |
| 美学 | 构图、色彩、光影、风格统一 | Judge | |
| 主体质量 ⚑ | 人物/产品主体无变形、崩坏、多指、穿模 | Judge | |
| 文字质量 | 画面内文字清晰、无乱码、无错字 | 混合：OCR 结果可规则查错字/乱码，版式 Judge | |

## L3 时序层（Temporal Quality）

| 维度 | 定义 | 评测方式 | 需参照 |
|---|---|---|---|
| 人物一致性 ⚑ | 同一人物跨镜头外观一致 | Judge（基于跨镜头帧序列） | |
| 物体恒常性 | 物体不凭空出现/消失/变化 | Judge | |
| 运动自然 | 动作、物理运动自然，无诡异插值 | Judge（必须基于连续片段） | |
| 无闪烁 | 无帧间闪烁、抖动、颜色跳变 | 混合：帧间差分类算法可检，Judge 复核 | |

## L4 音频层（Audio Quality）

| 维度 | 定义 | 评测方式 | 需参照 |
|---|---|---|---|
| 音质 | 无爆音、底噪、电流声，响度合规 | 客观：LUFS、削波、信噪比 | |
| 语音准确 ⚑ | 配音内容正确、语言正确、发音正确 | 混合：ASR 语言检测客观，内容对照事实表，发音 Judge | 部分 |
| 音画同步 | 口型/动作/字幕与音频对齐 | Judge（基于局部片段） | |
| 音乐适配 | BGM 情绪、节奏与内容匹配，音量层次合理 | Judge | |

## L5 叙事层（Narrative）

| 维度 | 定义 | 评测方式 | 需参照 |
|---|---|---|---|
| 开头吸引力 | 前 3 秒是否建立观看动机（hook） | Judge | |
| 镜头逻辑 | 镜头顺序服务叙事，无跳跃断裂 | Judge | |
| 节奏 | 剪辑节奏与内容密度、平台观看习惯匹配 | Judge | |
| 信息层级 | 重点信息被视觉/听觉强调，主次分明 | Judge | |

## L6 商业层（Commercial Usability）

| 维度 | 定义 | 评测方式 | 需参照 |
|---|---|---|---|
| 品牌一致性 ⚑ | Logo、色板、字体、口吻符合品牌规范 | 混合：色板/Logo 可模板比对，口吻 Judge | ✓ |
| 卖点表达 | 核心卖点被有效传达而非仅出现 | Judge | ✓ |
| 平台适配 ⚑ | 时长、比例、字幕规范、安全区、平台内容政策 | 客观为主：ffprobe 时长/比例，字幕规则；内容政策 Judge | ✓ |
| 可交付性 | 达到无需返工即可进入真实生产流程的标准 | Judge | ✓ |

## 范围外

生成系统的成功率、稳定性、耗时、成本、可复现性属于生成链路（kuleshov）的运行监控指标，**不属于本 benchmark**——VideoBench 的输入永远是已经产出的视频文件。

## 与外部 benchmark 的映射

- VBench 的 16 维 ≈ 本体系 L2 + L3 的细化，可作为这两层客观指标的实现参考；
- UVE-Bench 的四类（Static / Temporal / Dynamic / Alignment）≈ L2 / L3 / L3 / L1，其中 Video-Text Alignment 只覆盖 L1 的"指令遵循"；
- L5 / L6 是公开 benchmark 均未覆盖的部分，也是 VideoBench 相对它们的核心增量。

## 与旧 7 维评分卡的映射

视频调优工作簿的 D1–D7 迁移关系：D1 意图匹配→L1 指令遵循；D2 信息准确→L1 信息准确；D3 开头吸引力→L5；D4 节奏→L5；D5 音画协调→L4；D6 字幕版式→L2 文字质量 + L6 平台适配；D7 整体完成度→拆散，由 Gate + L6 可交付性替代（不再保留笼统总分维度）。
