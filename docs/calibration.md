# 人类黄金集校准与 Judge 有效性验证

## 为什么需要

AI Judge 逼近的是目标人群的判断分布，不是绝对真理。没有人类锚点的 Judge 分数只是模型的自言自语。

> 人类定义什么叫好，AI 负责把这个定义稳定地执行一万次。

## 校准闭环（六步）

1. **人工标注**：专业人员对一批样本对标注 G/S/B（附维度级理由），存入 `goldenset/annotations/`；
2. **校准 Judge**：用这批数据调整 Judge prompt、证据配置、聚合规则（改动必须走 [evolution.md](evolution.md) 的单变量 + 棘轮流程）；
3. **测一致率**：AI 与人的一致率（整体 + 分维度 + 分场景）；
4. **误判分析**：找出 AI 最容易误判的场景（如：字幕小错、品牌色偏差、节奏类差异），针对性补证据或补规则；
5. **放量**：让 AI 负责大规模评估；
6. **回流**：低置信度、换位冲突、高风险（Gate 边缘）样本交回人工，人工结论回流进黄金集。

## Judge 有效性的六项检验

一个 AI Judge 是否有效，看：

| 检验 | 度量 |
|---|---|
| 与专家判断的一致率 | GSB 三分类准确率 / Cohen's Kappa |
| 与普通用户偏好的相关性 | 与投放/观看数据或众包偏好的相关系数 |
| 重复评估稳定性 | 同一样本多次采样的结论一致率 |
| 换位一致性 | A/B 交换位置后结论一致率 |
| 致命问题识别率 | Gate 对人工标注致命错误的召回率（此项要求最高，漏放致命=最重故障） |
| 证据可验证性 | 抽查 Judge 引用的时间戳/引文是否真实存在 |

参考锚点：UVE-Bench 人类标注 Fleiss' Kappa 0.803、人类 pairwise 准确率 88%，强 MLLM 直接成对比较仅 ~51%——**pairwise 直接比较对模型远比单视频打分难**，所以 GSB Judge 必须经过本文件的校准流程才可信，且分维度比较（先拆维度再综合）优于让模型一步给出整体 GSB。

## 标注规范

- 每对样本 ≥ 2 名标注员，分歧样本仲裁后入库；
- 标注时同样先过 Gate、再分维度、最后 GSB，与 AI 同构，保证可比；
- 标注界面随机化 A/B 位置；
- 定期抽已标样本重标，监控标注标准漂移（沿用 video-score skill 的防漂移做法：分差超 1 档即重新对齐锚点）。

## 黄金集构成：类型 × 档位矩阵（2026-07-15 修订）

黄金集必须同时防两种失效——**错杀**（把顶级判成差）和**放水**（把差判成优）。只有好样本测不出放水，只有差样本测不出错杀。2026-07-15 试跑的教训：5 条顶级 showcase 让我们发现了口播口径的错杀，但换上宽容口径后是否放水，当时的黄金集完全无法回答。

构成要求：

- **类型维度**：覆盖全部业务类型（口播带货 / 知识解说 / showcase / MG 动画 / 品牌片……），每个类型独立成组——跨类型的分数没有可比性；
- **档位维度**：每个类型组内至少包含 顶级 / 中游 / 差 三档人类标注样本，外加各类致命错误样本（保证 Gate 召回可测）；
- **困难对**："接近 Same" 的成对样本（微小差异、互有胜负），这是 GSB 最容易翻车的区间；
- 现有 GoldenSet清单（视频调优工作簿）与 `annotations/2026-07-15-hyperframes-showcase-human.md`（showcase 类型组的第一批顶级样本）迁入/已入库。

## 双向回归指标

每次 Judge prompt / rubric / 模型改动，黄金集回归必须同时报告：

- **错杀率**：人类标注 顶级/优秀 的样本被判为 及格以下 的比例；
- **放水率**：人类标注 差/致命 的样本被判为 优秀以上 的比例；
- 加上原有的 GSB 三分类一致率与 Gate 致命召回率。

**任何一项退步即回滚**（evolution.md 棘轮）。特别警惕用引导性措辞修复错杀（如直接告诉 Judge"X 不是缺陷"）——它很容易把错杀翻转成放水，而放水在缺负样本的黄金集上是隐形的。
